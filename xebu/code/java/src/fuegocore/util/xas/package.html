Utilities for handling XML in programs. This package contains a generic event-sequence-based framework for programmatic use of XML. The intent is to support transparently through the same interfaces both reading and writing XML Infoset documents, regardless of how they are encoded as byte sequences.

The underlying interfaces that the framework expects are the XmlPull org.xmlpull.v1.XmlPullParser for input and org.xmlpull.v1.XmlSerializer for output. These interfaces need to be implemented by new serialization formats.

In actuality the output interface to implement is an extension of the XmlPull one, {@link fuegocore.util.xas.TypedXmlSerializer}, which provides a method for outputting typed content. The corresponding view on the parsing side is asymmetrically the {@link fuegocore.util.xas.TypedEventStream}. The best example source of using this feature is in the XAS unit test class {@link fuegocore.util.tests.XasTest}.
